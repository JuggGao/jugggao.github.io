curl

1 查看网页源码

1curl www.sina.com

2 自动跳转

默认情况下，curl 不会发送 HTTP 重定向，需要使用 -L 参数将请求重定向到新的地址上。

1curl -L www.sina.com

3 显示头信息

  • -i：显示 http response 的头信息，连同网页代码一起

  • -I：只显示头信息

1curl -i www.sina.com
2
3curl -I www.sina.com

4 下载文件

  • -o：下载文件时指定文件名（可包含目录）

  • -O：下载文件时使用 URL 中默认的文件名保存至当前目录

1curl -o /opt/gettext.html http://www.gnu.org/software/gettext/manual/gettext.html
2
3curl -O http://www.gnu.org/software/gettext/manual/gettext.html

  • --progress：显示进度条

  • -C：断点续传

1# 在文件下载完之前结束该进程
2curl -O http://www.gnu.org/software/gettext/manual/gettext.html --progress
3
4# 通过 -C 选项继续对该文件进行下载，已经下载的文件不会重新下载
5curl -C - -O http://www.gnu.org/software/gettext/manual/gettext.html --progress
  • --limit-rate：网络限速
1curl --limit-rate 10K -O http://www.gnu.org/software/gettext/manual/gettext.html

5 认证授权

访问需要授权的页面，可通过 -u 参数提供用户名和密码。

1curl -u username:password URL

6 FTP 下载上传

FTP 下载：

1# 列出 FTP 目录下文件目录列表
2curl -u user:password ftp://ftpserver
3
4curl ftp://user:password@ftpserver
5
6# 下载 FTP 文件
7curl -u user:password ftp://ftpserver/dir/gettext.html -o /tmp/gettext.html

FTP 上传：

1# 上传文件
2curl -u user:password -T gettext.html ftp://ftpserver/dir/
3
4# 修改文件内容或创建文件并添加内容
5echo "test" | curl -u admin:Ambow99999999 -T - ftp://10.10.113.39/admin/test.html

使用数组方式下载上传文件：

1curl –u user:password ftp://ftpserver/img/[1-10].gif –O
2
3curl –u user:password ftp://ftpserver/img/{one,two,three}.gif –O
4
5curl -u user:password -T "{file1,file2}" ftp://ftpserver/

7 设置代理

  • -x <host:port>：添加代理功能，同时支持 http、socks5 等协议

  • -U：指定代理的用户名密码

1curl -x proxyserver:port https://storage.googleapis.com

8 显示通信过程

  • -v：显示通信的整个过程

  • --trace <log-file>：显示更详细的通信过程并存储到指定文件中

1curl -v www.sina.com
2
3curl --trace output.log www.sina.com
4
5curl --trace-ascii output.log www.sina.com

9 cookie 信息保存使用

  • --cookie：发送 cookie

  • -c <cookie-file>：保存服务器返回的 cookie 到文件

  • -d <cookie-file>：使用保存的 cookie 文件

1curl --cookie "name=xxx" www.example.com
2
3curl -c cookies http://example.com
4
5curl -b cookies http://example.com

10 指定 HTTP 请求方法

curl 默认的请求方法为 GET，使用 -X 可以支持其他方法。

1curl -X POST www.example.com
2
3curl -X DELETE www.example.com

GET 方法相对简单，只要把数据附在网址后面就行：

1curl example.com/form.cgi?data=xxx

使用 POST 请求时，需要使用 --data 参数将数据提交上去：

1curl -X POST --data "data=xxx" example.com/form.cgi

如果你的数据没有经过 HTML 表单编码，还可以使用 --data-urlencode 参数进行编码：

1# 如果使用 --data 需要将空格转换为 %20
2curl -X POST --data "date=April%201" example.com/form.cgi
3
4# 使用 --data-urlencode 可以自动转码
5curl -X POST --data-urlencode "date=April 1" example.com/form.cgi

11 指定超时时间

  • --connect-timeout <seconds>：指定访问的超时时间
1curl --connect-timeout 3 www.sina.com

12 HTTPS 忽略证书验证

请求 HTTPS 协议时，可以添加 --inscure 或者 -k 来忽略证书验证。

1curl --insecure https://127.0.0.1

13 静默模式

  • -s：不输出统计信息

  • -S：与 -s 一起使用，如果失败 curl 将显示错误信息

1curl -s www.sina.com
2
3curl -sS https://storage.googleapis.com

可以结合 -o /dev/null 使其没有任何输出：

1curl -o /dev/null -s www.sina.com

14 输出格式化

变量说明
content_type内容类型
filename_effective写入的文件名
ftp_entry_path登录至 FTP 的初始路径
http_codeHTTP 状态码
http_connect上一次请求的响应码
local_ip发起请求的 IP 地址
local_port发起请求的端口号
num_connects传输过程中建立连接的数量
num_redirects请求过程中重定向的次数
redirect_url如果请求中有重定向但没有指定 -L 参数，此变量将显示重定向指向的实际 URL
remote_ip请求的远程 IP 地址
remote_port请求的远程端口号
size_download下载的总字节数
size_header下载的头字节数
size_request请求的字节数
size_upload上传的字节数
speed_download下载速度，单位 Bytes/s
speed_upload上传速度，单位 Bytes/s
ssl_verify_resultSSL 认证结果，0 表示认证成功
time_appconnect连接建立完成的时间，如 SSL/SSH 等建立连接或者完成三次握手时间
time_connect连接时间，从开始到建立 TCP 连接完成所用时间, 包括 DNS 解析时间，如果需要单纯的得到连接时间，用这个 time_connect 时间减去 time_namelookup 时间
time_namelookupDNS 解析时间，从请求开始到 DNS 解析完毕所用时间
time_pretransfer从开始到准备传输的时间
time_redirect重定向时间，包括到最后一次传输前的几次重定向的 DNS 解析、连接、预传输、传输时间
time_starttransfer开始传输时间，在发出请求之后，Web 服务器返回数据的第一个字节所用的时间，包括 time_pretransfer 和服务器需要的运算结果的时间
time_total总时间，时间单位全部按秒计
url_effective最终获取的 url 地址，尤其是当你指定给 curl 的地址存在 301 跳转，且通过 -L 继续追踪的情形
 1curl -o /dev/null -s -L -w "状态码: "%{http_code}\
 2"\n发起端（IP:Port）: "%{local_ip}:%{local_port}\
 3"\n远程端（IP:Port）: "%{remote_ip}:%{remote_port}\
 4"\n下载总大小: "%{size_download}\
 5"\n下载速度: "%{speed_download}\
 6"\n连接时间: "%{time_connect}\
 7"\n开始传输时间: "%{time_starttransfer}\
 8"\n重定向时间: "%{time_redirect}\
 9"\n总时间: "%{time_total}\
10"\n最终请求地址: "%{url_effective}"\n" \
11www.sina.com
